# Model Core Implementation Plan — Dynamic State-Space Dixon-Coles (NumPyro)

> The model code is intricate and fitting is iterative: each task is TDD'd on **synthetic data with known parameters** (recover them), then validated on the **real** international cache. Expect to refine against real data (as the clubform plan did).

**Goal:** Build `cupcast.v2.model` — a dynamic state-space Dixon-Coles goal model in NumPyro: each team's attack/defence evolve as a quarterly Gaussian random walk (form), the initial strength level is a learned function of the clubform composites, fitted by SVI (primary) or NUTS, exposing team ratings and per-match W/D/L + expected-goals + score-matrix predictions for the simulator.

**Architecture:** A NumPyro model (`model.py`) factoring the bivariate-Poisson/Dixon-Coles likelihood over an assembled international match table; a non-centered quarterly random walk on `attack_i`/`defence_i`; a learned linear `f_θ` prior from clubform; fitters (`fit.py`) returning a posterior; and a prediction layer (`predict.py`) producing W/D/L, xG, and a score matrix consumed by Plan 5. penaltyblog provides a static-DC baseline.

**Tech Stack:** Python ≥3.12, NumPyro 0.21 + JAX (CPU), numpy/pandas/scipy, penaltyblog, pytest.

## Locked modeling decisions (rationale for the papers)

- **Likelihood:** `home_goals ~ Poisson(λ)`, `away_goals ~ Poisson(ν)` with the Dixon-Coles low-score dependence correction `τ(x,y;λ,ν,ρ)` applied via `numpyro.factor`. `log λ = μ + α_h(p) − δ_a(p) + γ·host`, `log ν = μ + α_a(p) − δ_h(p)` (p = match period).
- **Dynamics (form):** for each team i and quarter t, `α_i[t] = α_i[t-1] + σ_α·z^α_i[t]`, `z ~ N(0,1)` (non-centered GRW); same for `δ` with `σ_δ`. The innovation SDs `σ_α, σ_δ ~ HalfNormal` are the principled replacement for v1's fixed decay.
- **Prior level (`f_θ`):** `α_i[0] ~ N(a0 + a1·clubform_attack_i, τ_prior²)`, `δ_i[0] ~ N(d0 + d1·clubform_defence_i, τ_prior²)`; coefficients `a0,a1,d0,d1` are learned. Data-poor teams stay near this prior; data-rich teams' walks move away. (`g_θ` style-matchup term deferred to a follow-on.)
- **Identifiability:** `μ` gets a tight prior at the empirical mean log scoring rate; the clubform prior fixes the team-strength level; `ρ ∈ (−0.3, 0.3)`.
- **Fitting:** SVI with `AutoNormal` (scalable to all teams×quarters; primary for validation + the Plan-7 tuner) and NUTS (final/validation on smaller cutoffs). Seeded.

### Task 1: International match table + static Dixon-Coles likelihood in NumPyro

**Files:** Create `src/cupcast/v2/model/__init__.py`, `src/cupcast/v2/model/matches.py`, `src/cupcast/v2/model/dixon_coles.py`. Tests `tests/v2/test_model_matches.py`, `tests/v2/test_dc_static.py`.

**Interfaces:**
- `matches.assemble_internationals(client, comp_seasons: list[tuple[int,int]], cutoff: str | None = None) -> pd.DataFrame` — load cached international fixtures, keep finished matches, return `date, home, away, home_goals, away_goals, competition, period, neutral, host_home`. `neutral = competition_id ∈ {1,4,6,7,9,22}` (finals tournaments at single hosts) else False; `host_home = ~neutral`. `period` = quarter index from the earliest date. If `cutoff` given, drop matches on/after it (holds out WC2026).
- `dixon_coles.dc_log_tau(x, y, lam, nu, rho)` — the DC correction (vectorized; mirror the proven v1 form in `src/cupcast/v1/model/dixon_coles.py` but reimplement). `dixon_coles.static_dc_model(home_idx, away_idx, host, x, y, n_teams)` — a NumPyro model with global `mu, gamma, rho`, per-team `attack, defense` (sum-to-zero via centered Normal), Poisson goals + `numpyro.factor("dc", dc_log_tau(...).sum())`.

- [ ] **Step 1: failing tests** — (a) `assemble_internationals` on a small in-memory fixture set yields the right columns, the neutral/host flags, and the cutoff filter; (b) fit `static_dc_model` with NUTS on SYNTHETIC data generated from known attack/defense and assert the posterior recovers the ranking (strongest team has highest posterior attack). Keep synthetic small (≤8 teams, ≤200 matches, num_warmup=200).
- [ ] **Steps 2–6:** fail → implement → pass → suite/lint/clean-room → stage. Then a real-data check (controller): assemble the full international table, fit static DC via SVI, print top-10 teams by `attack − defense` (sanity: Argentina/France/Spain/Brazil high).

### Task 2: Quarterly random-walk dynamics (the state-space)

**Files:** Create `src/cupcast/v2/model/dynamic.py`. Test `tests/v2/test_dynamic.py`.

**Interfaces:**
- `dynamic.dynamic_dc_model(home_idx, away_idx, period, host, x, y, n_teams, n_periods, attack_prior_loc=None, defense_prior_loc=None)` — non-centered GRW: `z_attack[i,t] ~ N(0,1)`, `attack[i,t] = attack0[i] + σ_α·cumsum_t(z)`; `attack0[i] ~ N(attack_prior_loc[i] or 0, τ_prior)`; same for defense; `σ_α, σ_δ ~ HalfNormal`. Indexes `attack[home_idx, period]` per match. Poisson + DC factor.

- [ ] **Step 1: failing test** — generate SYNTHETIC data where one team's attack TRENDS UP over periods; fit (SVI, ≥1000 steps, or short NUTS); assert the recovered `attack[i,t]` increases with t (form is captured) and `σ_α` is positive. Small (≤6 teams, ≤8 periods).
- [ ] **Steps 2–6:** fail → implement → pass → suite/lint/clean-room → stage; real-data check: fit dynamic DC via SVI on internationals, plot/print a known team's attack trajectory across quarters (sanity: reflects form swings).

### Task 3: clubform-informed learned prior (`f_θ`)

**Files:** Create `src/cupcast/v2/model/prior.py`. Test `tests/v2/test_prior.py`.

**Interfaces:**
- `prior.clubform_prior_locs(clubform: pd.DataFrame, teams: tuple[str,...]) -> tuple[np.ndarray, np.ndarray]` — align clubform `attack`/`defense` composites to the model's team index, standardized; NaN (no clubform) → 0.
- `prior.dynamic_dc_with_prior(...)` — extends `dynamic_dc_model` so `attack0[i] ~ N(a0 + a1·clubform_attack_i, τ_prior)` with learned `a0,a1,d0,d1 ~ N(0, ·)` (the `f_θ` linear map). Returns the same observed model.

- [ ] **Step 1: failing test** — synthetic data where teams with high clubform_attack truly score more; fit; assert learned `a1 > 0` (clubform predicts attack) and that a DATA-POOR team's posterior attack is pulled toward its clubform prior. 
- [ ] **Steps 2–6:** fail → implement → pass → suite/lint/clean-room → stage; real-data check: confirm `a1,d1` posteriors are positive and credible, and data-poor minnows lean on clubform.

### Task 4: Fit + predict interface + penaltyblog baseline

**Files:** Create `src/cupcast/v2/model/fit.py`, `src/cupcast/v2/model/predict.py`, `src/cupcast/v2/model/baseline.py`. Tests `tests/v2/test_fit_predict.py`, `tests/v2/test_baseline.py`.

**Interfaces:**
- `fit.fit_svi(model, data, seed=2026, steps=3000) -> Posterior` and `fit.fit_nuts(model, data, seed=2026, num_warmup=1000, num_samples=1000) -> Posterior`. `Posterior` wraps team index, period count, and posterior samples/means of `attack[:,T], defense[:,T], mu, gamma, rho`.
- `predict.match_score_matrix(posterior, home, away, host_home, max_goals=10) -> np.ndarray` (DC-corrected, posterior-mean rates). `predict.match_outcome(posterior, home, away, host_home) -> (p_home, p_draw, p_away)`; `predict.expected_goals(...)`. These mirror the v1 `DixonColesFit` API so Plan 5's simulator can consume either.
- `baseline.fit_penaltyblog_dc(matches) -> object` + `baseline.outcome_probs(...)` — a static weighted DC via penaltyblog, the comparison floor.

- [ ] **Step 1–6:** failing tests (predict on a tiny known posterior gives a valid probability simplex summing to 1, score matrix normalized; penaltyblog baseline returns sane probs) → implement → pass → suite/lint/clean-room → stage.

### Task 5: Real-data validation (ratings, calibration, vs baselines)

**Files:** Create `src/cupcast/v2/model/validate.py` (pure helpers + a `python -m` report). Test `tests/v2/test_model_validate.py`.

**Interfaces:**
