# Simulation Implementation Plan — 48-team Monte Carlo bracket

Build `cupcast.v2.sim`: a seeded Monte-Carlo simulator of the full 2026 World Cup
(12 groups → Round of 32 → R16 → QF → SF → final + third place) that consumes the v2
model's `predict` layer and produces per-team tournament probabilities. Mirrors the proven
v1 simulator (`src/cupcast/v1/sim/`) — reimplemented clean-room, adapted to the v2 model.

**Architecture:** `structure.py` (the real 2026 draw with API-Football names + the official
R32/R16/QF/SF/final templates + best-third slot rules), `group_stage.py` (sample group
scorelines from `predict.score_matrix`, FIFA tiebreakers), `bracket.py` (Annex-C best-thirds
allocation), `knockout.py` (regulation + extra time + shootout), `monte_carlo.py` (the 50k
driver → probabilities). A small extension to `cupcast.v2.model.predict` adds `host_away`.

**Tech Stack:** Python ≥3.12, numpy, pandas, pytest. Consumes `cupcast.v2.model.predict` + a `Posterior`.

## Reference (read, do NOT import): `src/cupcast/v1/sim/{worldcup2026,bracket,group_stage,knockout,monte_carlo}.py`

These encode the proven logic to mirror: the group/bracket templates, the FIFA tiebreaker
cascade (points → GD → GF → head-to-head → drawing of lots), the Annex-C backtracking
allocation of the 8 best thirds, the extra-time rate rescale + GK-adjusted shootout, and the
vectorised group/knockout sampling. Reimplement them in `cupcast.v2.sim` using v2's interfaces.

## Global Constraints

- **NEVER `git commit`/`git push`.** Stage with `git add` only; the user commits.
- **v2 code must NEVER import `cupcast.v1`** (clean-room; `tests/test_v2_namespace.py` guards it).
- **Every stochastic step is seeded** (`np.random.default_rng(seed)`); results reproduce exactly.
- Package/paths lowercase `cupcast`. Read only from cache. Tests use small in-memory inputs, no network/key.
- Full suite stays green after every task (starts at 255).

## Key adaptations from v1

1. **Team names = API-Football spellings** (the model's names): `USA` (not "United States"),
   `Czechia`, `Türkiye`, `Cape Verde Islands`, `Congo DR`, `Bosnia & Herzegovina`,
   `Curaçao`, `South Korea`. Everything else matches v1's draw.
2. **Score source:** v1 calls `fit.score_matrix(home, away, host_home, host_away, max_goals, rate_scale)`.
   v2 uses `predict.score_matrix(posterior, home, away, host_home, host_away, max_goals, rate_scale)`
   — which requires the `host_away` extension below.
3. **Host advantage:** hosts are `("Mexico", "USA", "Canada")`; in the group stage a host gets the
   boost in all three matches; in knockouts the venue country drives it. (Plan 6's venue/round-aware
   refinement comes later; here, mirror v1's team/venue host flag.)

### Task 1: Extend `predict` for `host_away` + tournament structure

**Files:** Modify `src/cupcast/v2/model/predict.py` and `src/cupcast/v2/model/fit.py` (Posterior.rate);
create `src/cupcast/v2/sim/__init__.py`, `src/cupcast/v2/sim/structure.py`. Tests:
`tests/v2/test_sim_structure.py`, extend `tests/v2/test_fit_predict.py`.

- **`Posterior.rate(home, away, host_home, host_away=False)`** — add the away-host term:
  `nu = exp(mu + attack[a] - defense[h] + gamma*host_away)`. **`predict.score_matrix` /
  `outcome_probs` / `expected_goals`** gain a `host_away: bool = False` parameter threaded through.
  (Backward-compatible default `False`.)
- **`structure.py`** mirrors v1's `worldcup2026.py` with API-Football names:
  - `GROUPS: dict[str, tuple[str,str,str,str]]` — the 12 groups A–L of the real 2026 draw
    (use the API-Football names per the mapping above).
  - `HOST_COUNTRIES = ("Mexico", "USA", "Canada")`.
  - `R32` / `R16` / `QUARTERFINALS` / `SEMIFINALS` / `FINAL` / `THIRD_PLACE` — copy v1's templates
    verbatim (they reference group letters + match numbers + venue countries, no team names).
  - `ALL_TEAMS`, `TEAM_GROUP`, `GROUP_LETTERS`.
  - `derive_groups_from_fixtures(client) -> dict[frozenset[str], None]` (or a list of 12 frozensets)
    — cluster the cached WC2026 (league 1, season 2026) group-stage fixtures into K4 components
    (teams that play each other are in one group). Used by the test to VERIFY `GROUPS` matches reality.

- [ ] **Step 1:** failing tests — (a) `Posterior.rate(..., host_away=True)` raises `nu` vs `host_away=False`;
  `score_matrix` host_away shifts probability toward the away team. (b) `structure.GROUPS` has 12 groups ×
  4 teams = 48 unique teams; every `R32` slot references a valid group; the derived groups from a small
  in-memory fixture set match the expected composition.
- [ ] **Step 2–6:** fail → implement → pass → suite/lint/clean-room → stage. **Controller real-data check:**
  `derive_groups_from_fixtures(real_client)` matches `structure.GROUPS` exactly (the hardcoded draw is correct).

### Task 5: Monte-Carlo driver + tournament probabilities

**Files:** `src/cupcast/v2/sim/monte_carlo.py`. Test `tests/v2/test_sim_monte_carlo.py`.

- [ ] **Step 1:** failing test — `simulate_tournament(stub_posterior, n_sims=200, seed=1)` returns a

- [ ] **Step 2–6:** fail → implement → pass → suite/lint/clean-room → stage. **Controller real-data

  championship probabilities (sanity: elite teams on top, host bumps plausible).
