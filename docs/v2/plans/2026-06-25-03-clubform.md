# Clubform Subsystem Implementation Plan

>
> **Execution is deferred until the v2 data pull completes** (it populates `data/raw/api_football/players`, `.../fixtures`, `.../fixtures/lineups`). Tasks are built and unit-tested on in-memory fixtures shaped like the real cached data; an end-of-plan smoke validates against the real cache.

**Goal:** Build `cupcast.v2.clubform` — a separate-semantic subsystem that turns cached club player stats, international lineups, and ClubElo into per-team, position-aware, league-strength-adjusted squad profiles (attack / defence / goalkeeping + a formation/style profile), weighted by data-driven expected tournament minutes, for the model core (Plan 4) to consume.

**Architecture:** Four layers, each a focused module: (1) a ClubElo-based league-strength index; (2) a position-aware player-quality parser over API-Football season stats (Understat npxG+xA as an attack override where available); (3) expected national-team minutes derived from real recent international lineups; (4) a squad-composite assembler producing per-team profiles. The subsystem reads only from cache and never imports `cupcast.v1`.

**Tech Stack:** Python ≥3.12, pandas, numpy, `soccerdata` (ClubElo, already a dependency), pytest.

## Locked modeling decisions (from design + brainstorm)

- **League strength:** normalized median ClubElo of a league's clubs; a player's per-90 quality is scaled by their club league's index. Performance-only, no market values.
- **Player quality:** position-aware multi-metric — ATT (npxG+xA where available, else goals+assists+shots-on+key-passes per 90), DEF (tackles+interceptions+blocks+duels-won per 90), GK (save rate + goals-conceded per 90). z-scored within position group. API-Football `games.rating` is a position-agnostic anchor.
- **Expected minutes:** per-player starter probability × expected minutes estimated from recent real national-team lineups (start frequency + minutes), not depth-chart heuristics.

## Confirmed data shapes (from the live cache)

- `/players` stat block: `games{minutes,position,rating,appearences,lineups}`, `goals{total,assists,saves,conceded}`, `shots{total,on}`, `passes{total,key}`, `tackles{total,blocks,interceptions}`, `duels{total,won}`. `position ∈ {Goalkeeper, Defender, Midfielder, Attacker}`. No xG fields.
- `/fixtures/lineups` per team: `team{id,name}`, `formation` (e.g. "4-2-3-1"), `startXI[{player{id,name,pos,grid}}]`, `substitutes[...]`.
- ClubElo (via `soccerdata.ClubElo`): per-club `elo`, `country`, `level`, date range.

## Global Constraints

- **NEVER `git commit`/`git push`.** Each task ends by staging with `git add`; the user commits.
- **v2 code must NEVER import `cupcast.v1`** (clean-room; enforced by `tests/test_v2_namespace.py`). Mirror v1's `expected_minutes`/`squad_strength` patterns by reimplementation, not import.
- Package/paths stay lowercase `cupcast`. **No market-value / Transfermarkt data** — performance metrics only.
- Read only from cache; tests use in-memory fixtures, never the network or a key.
- `data/` is never committed. Full suite stays green after every task (starts at 75).

### Task 1: League-strength index (ClubElo)

**Files:** Create `src/cupcast/v2/clubform/__init__.py`, `src/cupcast/v2/clubform/league_strength.py`. Test `tests/v2/test_league_strength.py`. (ClubElo fetch cached under `data/raw/clubelo/`.)

**Interfaces:**
- `fetch_clubelo(cache_dir="data/raw/clubelo") -> pd.DataFrame` — via `soccerdata.ClubElo`, cache-first; columns include `team`, `elo`, `country`, `level`.
- `league_strength_index(clubelo: pd.DataFrame, club_to_league: dict[str, int]) -> dict[int, float]` — for each league id, the normalized (z then min-max to ~[0.5, 1.5]) **median** latest ClubElo of its clubs; leagues with <3 matched clubs fall back to the global median (index 1.0).
- `normalize_club_name(name: str) -> str` — accent/case-fold for matching ClubElo `team` to API-Football club names.

**Approach note (finalize on real data):** the `club_to_league` map is derived from the fetched fixtures' team lists per league id (Task in Plan 2 cached these). ClubElo team names and API-Football names differ; `normalize_club_name` + a small alias table closes the gap. The end-of-plan smoke reports unmatched-club coverage per league so the alias table can be tuned.

### Task 2: Position-aware player-quality parser

**Files:** Create `src/cupcast/v2/clubform/player_quality.py`. Test `tests/v2/test_player_quality.py`.

**Interfaces:**
- `parse_player_stats(players_response: list[dict], league_id: int, min_minutes: int = 450) -> pd.DataFrame` — one row per player with `player_id, name, position, minutes, rating, att_raw, def_raw, gk_raw`, dropping players under `min_minutes`. Raw per-90 definitions:
  - `att_raw = (goals.total + goals.assists) / m90 + 0.5*shots.on/m90 + 0.3*passes.key/m90`
  - `def_raw = (tackles.total + tackles.interceptions + tackles.blocks + duels.won) / m90`
  - `gk_raw  = goals.saves / m90` and `save_rate = saves / (saves + goals.conceded)` (NaN-safe) for Goalkeepers; non-GKs get `gk_raw = NaN`.
  - `m90 = minutes / 90`.
- `apply_league_strength(quality: pd.DataFrame, league_id: int, strength_index: dict[int, float]) -> pd.DataFrame` — multiplies `att_raw, def_raw` (not GK save_rate) by `strength_index.get(league_id, 1.0)`.
- `zscore_within_position(quality: pd.DataFrame) -> pd.DataFrame` — adds `att_z, def_z, gk_z` z-scored within each `position` group (so a defender is rated vs defenders).
- (Override hook) `apply_understat_xg(quality, understat_df)` — where an Understat npxG+xA per-90 exists for a matched player, replace the attacking raw with it before z-scoring. Optional; no-op if Understat absent.

- [ ] **Step 1: Write failing tests** — feed a 2-player in-memory `/players` response (an attacker and a goalkeeper, shaped like the confirmed cache), assert per-90 metrics, the <450-minute drop, league-strength scaling, and within-position z-scores.
- [ ] **Step 2–6:** fail → implement → pass → suite/lint/clean-room → stage.

### Task 3: Expected national-team minutes from real lineups

**Files:** Create `src/cupcast/v2/clubform/expected_minutes.py`. Test `tests/v2/test_expected_minutes_v2.py`.

**Interfaces:**
- `national_team_appearances(lineups_by_fixture: dict[int, list[dict]], fixture_dates: dict[int, str], half_life_days: float = 540) -> pd.DataFrame` — from cached international lineups, per `player_id`: recency-weighted `starts`, `sub_apps`, and total weighted appearances (exponential decay by `half_life_days` on the fixture date).
- `expected_minutes(appearances: pd.DataFrame, full_match: float = 90.0) -> pd.DataFrame` — `start_prob = weighted_starts / weighted_team_matches`; `exp_minutes = start_prob * full_match + (1 - start_prob) * sub_minutes_prior`; clipped to [0, 90]. Returns `player_id, start_prob, exp_minutes`.

**Approach note:** a player's national team is inferred from the lineup's `team`; recency weighting mirrors the goal model's decay so "current form of selection" dominates. Coverage gaps (nations with sparse cached lineups) fall back to uniform expected minutes across the 26-man squad — reported by the smoke.

- [ ] **Step 1–6:** failing test (in-memory lineups for two fixtures with a recurring starter and a one-off sub; assert higher `start_prob`/`exp_minutes` for the starter) → implement → pass → suite/lint/clean-room → stage.

### Task 4: Squad-composite assembler + style profile

**Files:** Create `src/cupcast/v2/clubform/composite.py`. Test `tests/v2/test_composite.py`.

**Interfaces:**
- `squad_profiles(quality: pd.DataFrame, exp_minutes: pd.DataFrame, squads: pd.DataFrame) -> pd.DataFrame` — joins player quality (`att_z, def_z, gk_z`) to the 26-man squads and their `exp_minutes`; returns one row per national team with minutes-weighted `attack`, `defense`, `gk` composites and `coverage` (share of squad minutes matched to club data). Players unmatched to club data impute at the position-group 20th percentile (mirroring v1's floor).
- `style_profile(lineups_by_fixture, team_name) -> dict` — formation-usage distribution (e.g. `{"4-3-3": 0.6, "4-2-3-1": 0.4}`) and an outfield share by line (DEF/MID/ATT) from recent national-team lineups; the seed of the Plan-4 "system" matchup term (deep tactical metrics deferred to event data).
- `build_clubform(...) -> pd.DataFrame` — the top-level entry the model core consumes: per-team `attack, defense, gk, coverage, style`. Reads from cache via the Plan-2 client + `parse_player_stats`/`apply_league_strength`/`zscore_within_position`/`expected_minutes`.

- [ ] **Step 1–6:** failing test (small fixtures: two teams, a few players each with z-scores + expected minutes, assert minutes-weighted composites, coverage, and the position-floor imputation) → implement → pass → suite/lint/clean-room → stage.
