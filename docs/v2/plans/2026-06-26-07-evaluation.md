# Evaluation & Tuning Implementation Plan

Build `cupcast.v2.validate` (eval) + `cupcast.v2.tune` (Optuna): a reproducible evaluation
harness that scores the held-out v2 forecast against the actual WC2026 results head-to-head
with v1, the market, and uniform; a rolling-origin Optuna hyperparameter tuner; and the
hook for the pre/post-LLM ablation (Plan 6). This formalises the one-off analysis that
showed v2 (log-loss 0.856) beats v1 (0.869) on the 60 played WC2026 group matches.

**Architecture:** `wc2026.py` (assemble the actual WC2026 results + per-match held-out
predictions for v2/v1/market/uniform + score them), `tune.py` (Optuna study over the model
hyperparameters with a rolling-origin CV objective, seeded + checkpointed), reusing the
`cupcast.v2.model.validate` scorers (log_loss/Brier/RPS).

**Tech Stack:** Python ≥3.12, NumPyro model, Optuna, numpy/pandas, pytest.

## Global Constraints

- **NEVER `git commit`/`git push`.** Stage with `git add` only; the user commits.
- **v2 code must NEVER import `cupcast.v1`** — EXCEPT the dedicated comparison module
  `cupcast.v2.compare` (see note), which is allowed to import v1 to score the baseline.
  (The clean-room guard covers `src/cupcast/v2`; the v1-vs-v2 comparison lives in
  `src/cupcast/compare/`, outside the guard, and may import both.)
- **Every stochastic step seeded**; Optuna `TPESampler(seed=...)`, model fits seeded.
- WC2026 is the **held-out test set** — the tuner uses rolling-origin CV on pre-cutoff
  internationals ONLY; WC2026 is scored once, never tuned on.
- Tests use small synthetic/stub inputs; no network/key. Full suite green after each task (starts at 312).

## Note on the v1 comparison & clean-room

The held-out comparison must score v1 too, which means importing `cupcast.v1`. To keep the
v2 clean-room guard intact, the v1-vs-v2 scoring lives in `src/cupcast/compare/wc2026.py`
(the `cupcast.compare` package, which the guard does NOT cover). The pure v2 scoring +
Optuna tuner live under `src/cupcast/v2/`.

### Task 1: WC2026 actual-results assembler + held-out scoring (v2 + uniform)

**Files:** Create `src/cupcast/v2/validate/wc2026.py` (the file `cupcast/v2/validate/` is new —
add `__init__.py`). Test `tests/v2/test_wc2026_eval.py`.

**Interfaces:**
- `actual_wc2026_results(client) -> pd.DataFrame` — cached WC2026 (league 1, season 2026)
  finished matches with results; columns `home, away, home_goals, away_goals, outcome`
  (0=home,1=draw,2=away), `stage` (Group/R32/…). Filters by round string.
- `score_predictions(P: np.ndarray, y: np.ndarray) -> dict` — `{n, log_loss, brier, rps}`
  reusing `cupcast.v2.model.validate.{log_loss,brier,rps}`.
- `held_out_v2(posterior, results, hosts=("Mexico","USA","Canada")) -> tuple[np.ndarray, np.ndarray]`
  — per-match `predict.outcome_probs(posterior, home, away, host_home=home in hosts,
  host_away=away in hosts)` for matches whose both teams are in `posterior.teams`; returns
  `(P, y)`.

- [ ] **Step 1:** failing tests — `actual_wc2026_results` on a stub client returns the right
  rows/outcomes; `score_predictions` on a perfect prediction gives log_loss≈0; `held_out_v2`
  with a stub posterior returns aligned `(P, y)` and applies host flags. (no network)
- [ ] **Step 2–6:** fail → implement → pass → suite/lint/clean-room → stage. **Controller real
  run:** confirm it reproduces the one-off numbers (v2 log-loss ≈0.856 on the 60 played matches).

### Task 2: v1-vs-v2 comparison module (`cupcast.compare`)

**Files:** Create `src/cupcast/compare/wc2026.py`. Test `tests/test_compare_wc2026.py`.

**Interfaces:**
- `V1_NAME_MAP` — API-Football → v1 team names (`USA`→`United States`, `Czechia`→`Czech
  Republic`, `Türkiye`→`Turkey`, `Cape Verde Islands`→`Cape Verde`, `Congo DR`→`DR Congo`,
  `Bosnia & Herzegovina`→`Bosnia and Herzegovina`).
- `held_out_v1(v1_fit, results, hosts) -> tuple[np.ndarray, np.ndarray]` — maps names, calls
  `v1_fit.outcome_probs(home_v1, away_v1, host_home, host_away)` for matches whose both teams
  are in `v1_fit.teams`; returns `(P, y)`.
- `compare_table(client) -> pd.DataFrame` — builds the actual results, fits v1 (via
  `cupcast.v1.run.build_fit`) and v2 (via the model pipeline), scores v2/v1/uniform on the
  COMMON matches, returns a tidy table. This is the deliverable that reproduces the headline.

- [ ] **Step 1:** failing tests — `held_out_v1` with a stub v1-fit + the name map aligns rows;
  the comparison driver returns rows for v2/v1/uniform with the metric columns. (no network)
- [ ] **Step 2–6:** fail → implement → pass → suite/lint/clean-room (note: `cupcast.compare`
  is OUTSIDE the v2 guard, so importing v1 here is allowed) → stage. **Controller real run:**
  reproduce the v2-beats-v1 table on the live cache.

### Task 3: Optuna rolling-origin hyperparameter tuner

**Files:** Create `src/cupcast/v2/tune/__init__.py`, `src/cupcast/v2/tune/optuna_cv.py`.
Test `tests/v2/test_tune.py`. Adds the `optuna` dependency.

**Interfaces:**
- `rolling_origin_folds(matches, fold_dates) -> list[tuple[train_idx, test_idx]]` — time-respecting
  splits (train strictly before each fold date; test in `[fold, next_fold)`). Pure, unit-tested.
- `objective(trial, matches, clubform, fold_dates) -> float` — samples hyperparameters
  (`sigma_att/sigma_def` innovation SD priors, `tau_prior`, the SVI step count / learning rate,
  the period granularity in months), fits the model per fold via fast SVI, predicts the fold's
  test matches, returns mean OOS log-loss across folds. Seeded.
- `tune(matches, clubform, fold_dates, n_trials=40, seed=2026, storage=None) -> optuna.Study` —
  runs the study (`TPESampler(seed=...)`, optional RDB/journal storage for checkpoint/resume).
  Prints the best params + OOS score. WC2026 is NEVER in `matches` (cutoff enforced upstream).

- [ ] **Step 1:** failing tests — `rolling_origin_folds` produces strictly time-ordered,
  non-overlapping splits with no leakage (max train date < fold date ≤ min test date); `objective`
  with a TINY synthetic problem + a stub fit returns a finite score; `tune` with `n_trials=2` on
  synthetic returns a study whose best value is finite and reproduces with the same seed. Keep
  fast (stub/cheap fits in tests).
- [ ] **Step 2–6:** fail → implement → pass → suite/lint/clean-room → stage. **Deferred overnight
  run:** `tune(...)` with `n_trials≈40` on the real internationals → best hyperparameters; refit
  the model with them; re-score the WC2026 hold-out (expect the v2 edge to widen).

### Task 4: Ablation hook + the evaluation report

**Files:** Create `src/cupcast/v2/validate/report.py` (a `python -m` eval report). Test
`tests/v2/test_eval_report.py`.

**Interfaces:**
- `ablation_score(results, posterior_with, posterior_without) -> pd.DataFrame` — scores the model
  WITH vs WITHOUT the context covariates on the held-out results (the pre/post-LLM ablation; the
  `posterior_without` is the same model fit with context coefficients zeroed — wired fully in
  Plan 6, here the interface + a stub-covariate test).
- `main()` — assemble the actual WC2026 results, fit v2 (cutoff), print the held-out scoring table
  (v2 / v1 / uniform via `cupcast.compare`) and, when a tuned config exists, the tuned-v2 row.
  Real-data smoke (not unit-tested).

- [ ] **Step 1–6:** unit-test `ablation_score` on stub posteriors (a model whose covariate makes
  predictions better scores better WITH than WITHOUT) → implement → pass → suite/lint/clean-room →
  stage. Run `python -m cupcast.v2.validate.report` and record the table.

  (Task 2, in `cupcast.compare` to preserve the v2 guard), Optuna rolling-origin CV tuning (Task 3),

- **Reproducibility:** every fit + the Optuna study are seeded; the tuner is checkpointable.
